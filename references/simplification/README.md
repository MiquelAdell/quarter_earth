# Workshop simplification — working docs

Helper documents for turning Half-Earth Socialism into a ~1-hour, multiplayer,
discussion-driven planning game (the acTe assembly-workshop use case, see
`../project-brief.md`).

We are using spec-driven development. The intended flow:

1. ~~`open-questions.md`~~ — **answered 2026-07-02**; folded into the spec.
2. ~~`open-questions-round2.md`~~ — **answered 2026-07-02**; folded into the
   spec and card proposal.
3. **`spec.md`** — **v1, decided.** Numbers subject to M1 playtesting.
4. **`policy-cards-proposal.md`** — **v1.1, accepted**; 21-card set with
   prerequisite phasing.
5. **`implementation-plan.md`** — work breakdown M1 → M3, with an execution
   log at the bottom. **Status: M1 done; M2 in progress (W4–W6 done, W7
   next).** See also `m1-run-sheet.md` (moderator guide) and
   `worlds/workshop-m1.world` (generated by `util/workshop_world.py`).

## Documents

| File | What it's for |
|------|---------------|
| `game-mechanics.md` | Inventory of the current game: turn loop, all decision types, currencies, feedback variables, content volume. Read this to know what exists before deciding what to cut. |
| `codebase-map.md` | Where each mechanic lives in the code, and how the pieces connect. Read this when assessing the cost of a change. |
| `simplification-levers.md` | Concrete simplification options, each with the mechanic it removes/changes, the code it touches, and an effort estimate. The menu to build the spec from. |
| `open-questions.md` | Round-1 decisions (answered). |
| `open-questions-round2.md` | Round-2 decisions. Answer inline. |
| `spec.md` | Workshop-mode spec, draft v0. |
| `policy-cards-proposal.md` | Proposed ~27-card set with rationale and costs. |

## Key constraints (from the project brief)

- Target: playable in ~1 hour by a group, moderator-led, decisions debated aloud.
- Discussions must be grounded in numbers with visible consequences.
- Cut: text volume, variable count, option count, decision-type count.
- Preferred decision model: **one decision type — policies**. Things that are
  currently other decision types (energy-mix allocation, research) get
  re-expressed as policy cards where kept (e.g. a "Wind push" policy that
  shifts electricity mix toward wind).
- Target event: October 2026. Upstream (Francis) may build a "model-only mode"
  but has no committed timeline — assume we prototype independently.
